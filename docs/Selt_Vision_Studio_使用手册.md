



## 1. 和 VisionMaster 的对照

| VisionMaster / 同类软件 | 本软件对应概念 |
|-------------------------|----------------|
| 方案 / Solution | **工程**（`.selt` 或 `.seltproject`） |
| 流程 / Procedure | **流程**（画布上方流程栏，可多流程） |
| 工具 / Tool | **算子节点**（左侧工具箱） |
| 工具连线 | **端口连线**（带数据类型，非法连线会被拒绝） |
| 全局变量 | **变量管理** + 参数**绑定** |
| ROI / 示教 | **图像与结果 → 结果** 页交互绘制 |
| 单次 / 连续运行 | **F5 单次** / **连续运行** / **F8 停止** |
| 运行到某模块 | **运行此节点** / **从此节点运行** |
| OK（带警告） | 节点状态 **成功(警告)**，流程可继续 |

上手时不必从零学「视觉」，只需熟悉本软件的界面分区与交互习惯。

## 2. 启动与安装

### 2.1 便携包（推荐给现场电脑）

将整个 `Selt_Gui` 文件夹拷到目标机器（不要只拷 `Selt_Gui.exe`），双击运行：

```text
Selt_Gui/
  Selt_Gui.exe
  Qt6*.dll / platforms/ / imageformats/ ...
  libopencv_*.dll
  plugins/nodes/          ← 可选节点插件
  README.txt
```

缺任意同级 DLL（尤其 OpenCV）都会弹「找不到 xxx.dll」。

### 2.2 开发机

用 Qt Creator 打开工程根目录 `CMakeLists.txt`，Kit 选 Qt 6 + MinGW，构建目标 `Selt_Gui`，运行 `build/.../bin/Selt_Gui.exe`。

## 3. 主界面一览

默认布局（深色工业主题）：

```text
┌──────────┬─────────────────────────────┬──────────────────┐
│  工具箱   │  流程栏（多流程切换）          │                  │
│  算子目录 │─────────────────────────────│   图像与结果      │
│          │                             │   · 编辑 / 结果  │
│  输入    │         流 程 画 布           │   · 结果清单     │
│  预处理  │      （节点 + 端口连线）       │   · 运行监视     │
│  定位…   │                             │   · 帮助         │
└──────────┴─────────────────────────────┴──────────────────┘
              顶部工具栏：新建/打开/保存 · 单次/连续/停止
```

| 区域 | 作用 |
|------|------|
| **工具箱** | 按阶段分类的算子：拖拽或双击添加到画布 |
| **流程栏** | 同一工程内多个流程：新建 / 重命名 / 删除 |
| **画布** | 编排 DAG；选中节点后右侧联动 |
| **图像与结果** | 看图、示教 ROI/几何、看测量结果与运行日志 |
| **检查器**（视图菜单打开） | **属性** / **输入源** / **说明** |
| **项目资源 / 通讯 / 变量** | 默认隐藏，按需从 **视图** 打开 |

**视图 → 恢复默认布局** 可一键回到上述结构。

## 4. 标准调试流程（5 分钟）

### 步骤 A：新建工程

1. **文件 → 新建**  
2. **文件 → 保存**，建议使用 `*.selt`  
3. 需要资源目录结构时，也可使用 `*.seltproject` 工程容器  

### 步骤 B：放一条最小链路

任选其一：

- **执行 → 插入流程模板**（如「基础测量流程」「单 Blob 检测」），再改图改参；或  
- 从工具箱拖入：**图片输入 → 预处理（可选）→ 定位/测量 → 判定（可选）→ 结果预览**

端口连线：从上游输出端口拖到下游输入端口。类型不匹配时无法连接。

### 步骤 C：指定图像

两种方式可并存，优先级以实际运行为准：

1. **图片输入**节点里填本地路径；或  
2. **视图 → 检查器 → 输入源**：  
   - 单张图片 / 目录回放 / 相机（索引 0–16）  
   - 打开 → 软触发  

若未打开输入源，运行时会使用流程里「图片输入」节点的路径。

### 步骤 D：示教 ROI / 查找几何

1. 选中需要示教的节点（如 Blob、查找直线、查找圆、卡尺）  
2. 切到 **图像与结果 → 结果**  
3. 在预览上拖拽 / 点击示教（见第 6 节）  
4. 参数会写回节点；交互示教会把对应参数标成 **用户设定**（不再被 Auto 覆盖）

### 步骤 E：运行与看结果

| 操作 | 快捷键 / 入口 |
|------|----------------|
| 单次跑完整流程 | **F5** 或工具栏「单次执行」 |
| 连续跑 | 工具栏「连续运行」 |
| 停止 | **F8** |
| 只跑当前节点 | 画布右键调试，或属性头 ▶ **运行此节点** |
| 从当前节点起跑（含上游） | **从此节点运行** |
| 断点 | **F9** |

看结果：

- 节点角标：运行中 / 成功 / **成功(警告)** / 失败  
- **结果清单**：测量值、单位、判定、置信度  
- **运行监视**：耗时、失败阶段、诊断码  
- **帮助**：当前节点的模块说明  

### 步骤 F：发布前检查（可选）

**执行 → 发布前校验**：检查缺输入、断线、明显配置错误，适合拷到产线前自检。

## 5. 算子分类速查

工具箱按流程阶段排序（与 VisionMaster 工具组类似）：

| 阶段 | 常见算子 |
|------|----------|
| **输入** | 图片输入、模板源 |
| **预处理** | 灰度、二值、滤波、边缘、形态学、缩放、CLAHE、锐化… |
| **区域** | 掩膜合成、轮廓提取、连通域、Blob 分析 |
| **定位** | Hough 线/圆、Blob 定位、特征匹配、多模板匹配 |
| **识别** | 条码/二维码、OCR |
| **测量** | 模板匹配、查找直线/圆、卡尺、拟合线/圆、距离/角度/平行/垂直度… |
| **判定** | 范围判定、容差判定、检测结果打包 |
| **逻辑** | 与或非、比较、条件选择、计数器、变量写入 |
| **通讯** | COM / TCP / Modbus 读写、编解码、帧封装… |
| **输出** | 结果预览、结果写出 |
| **流程** | 子流程 |

搜不到工具时：用工具箱顶部搜索框；确认已编译带 OpenCV 的版本（无 OpenCV 时仅编辑器外壳）。

## 6. 图像交互示教（重点）

在 **结果** 预览中操作（选中对应节点后）：

| 意图 | 常见操作 |
|------|----------|
| 矩形 ROI | 拖拽框选；属性里也可看到 ROI 参数 |
| 多边形 ROI | **Ctrl+点击**加点，**双击**结束 |
| 点 / 轮廓点 | 按节点提示点击示教 |
| 直线 / 圆 / 卡尺条带 | 按几何种类拖拽或两点定义 |
| 清除 | 预览工具栏「清除 ROI」等 |

习惯建议（与 VisionMaster 示教类似）：

1. 先 **F5** 跑出有图的结果页，再示教更直观。  
2. 示教后参数变为用户模式；若要恢复自动估计，在属性中把对应项改回 Auto / 复位默认。  
3. **Ctrl** 更偏向 ROI；部分几何用 **Shift** 重新示教（以界面提示为准）。

预览工具栏：**适应 / 1:1 / 缩放 / 棋盘格 / Overlay 过滤**。

## 7. 参数、绑定与 Auto

### 7.1 属性面板

选中节点 → **检查器 → 属性**（或从视图打开检查器）：

- 参数按分组显示（阈值、采样、ROI、拟合…）  
- **复位默认参数** 可撤销式恢复  
- 每行可设置**绑定**（链接图标）

### 7.2 绑定三种来源

打开绑定编辑器后可选：

1. **常量** — 面板直接填值  
2. **上游输出** — 引用前序节点端口（类似 VM 的链接输出）  
3. **项目变量** — 来自变量管理（全局 / 工程 / 当前流程）

绑定后运行时由解析器取值；改上游结果会自动驱动下游。

### 7.3 Auto 参数

部分阈值、卡尺数量、极性等支持 **Auto**：

- 未示教 / 未强制时，算法按图像与几何自动估计  
- 在图上示教或手动改值后，一般进入 **用户** 模式  
- Blob 极性等：白件黑底常用 `light` / `any`；默认若不合适请在属性中改，或重建节点清掉旧 JSON

## 8. 标定、资源与通讯

| 功能 | 入口 | 说明 |
|------|------|------|
| 标定 | **执行 → 标定管理** | 激活后长度类结果可带物理单位；未标定多为 `px` |
| 项目资源 | **视图 → 项目资源** | 模板图、标定、工程内图像等 |
| 变量 | **视图 → 变量管理** | 作用域：全局 → 工程 → 当前流程 |
| 通讯 | **视图 → 通讯配置** | TCP / COM / Modbus 配置档；节点里也可 Mock 联调 |

现场通讯请关闭 Mock，并确认便携包中带有 `Qt6Network`（及串口相关 DLL，若使用 COM/Modbus RTU）。

## 9. 端口显示（画布清洁）

完整端口在模块描述里都有，画布默认只显示**主图像链路**，避免像 VisionPro 那样端口过多。

需要连测量值、判定结果等到下游时：

1. 选中节点 → 属性 → **输入与输出**  
2. 勾选要暴露的端口  
3. 画布上出现端口胶囊后再连线  

隐藏端口不影响已有绑定与运行，只是画布上不显示。

## 10. 运行状态怎么读

| 状态 | 含义 | 流程是否继续 |
|------|------|----------------|
| 成功 | 正常完成 | 是 |
| **成功(警告)** | 有结果但质量偏低 / 软失败（如弱匹配） | **是**（可继续判定） |
| 失败 | 硬失败（无图、几何非法、超时等） | 否（后续依赖节点通常不跑） |

失败时优先看 **运行监视** 里的诊断码（缺输入、无效几何、质量拒绝等），再回属性与示教区修正。

## 11. 推荐调试顺序（产线习惯）

1. **图片输入** 能出图（结果页有图）  
2. 预处理出稳定对比度（必要时二值 / CLAHE）  
3. 定位或 Blob 出候选（注意极性、ROI）  
4. 测量几何示教（找线 / 找圆 / 卡尺）  
5. 判定容差（先像素，再上标定）  
6. 需要时接通讯写出 / 结果写出  
7. **发布前校验** → 连续运行压测  

卡住时：用 **运行此节点** 缩小范围，比整图 F5 更快定位。

## 13. 常见问题

**Q：双击 exe 提示缺少 `libopencv_xxx.dll`？**  
A：必须运行**整个便携目录**；把缺失的 OpenCV DLL 放到 exe 同级（重新执行 `scripts/package_windows.ps1` 最省事）。

**Q：Blob 检不到白目标？**  
A：检查极性（`dark` / `light` / `any`）与 ROI；旧节点可能残留错误默认，改属性或重建节点。

**Q：查找直线/圆总失败？**  
A：先在结果图示教搜索几何与卡尺区域；看是否变成「成功(警告)」；过严质量阈值可适当放宽。

**Q：相机列表为空？**  
A：确认便携包含 `libopencv_videoio*.dll` 与 ffmpeg 相关 DLL；驱动与索引正确；无相机时用文件/目录回放。

**Q：工具箱没有视觉算子？**  
A：当前构建未启用 OpenCV（`SELT_HAS_OPENCV=0`），仅为编辑器外壳。

**Q：布局乱了？**  
A：**视图 → 恢复默认布局**。

## 15. 学习路径建议

1. 插入模板 → F5 → 看结果清单（建立「节点图 = 流程」直觉）  
2. 改一张图路径 → 示教 ROI → 再跑（建立示教习惯）  
3. 做「定位 → 测量 → 容差判定」三条链  
4. 需要对外输出时再开通讯 / 变量绑定
