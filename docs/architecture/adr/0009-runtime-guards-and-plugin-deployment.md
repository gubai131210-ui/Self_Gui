# ADR-0009：运行时输入防护与插件部署诊断

## 状态

已接受。

## 决策

- 所有图像加载统一经 `ImageLoader`，使用 `QFile` + `cv::imdecode` 兼容 Windows 中文路径。
- OpenCV 算法在调用前必须先完成尺寸、位深、通道和 ROI 校验；`cv::Exception` 转为结构化诊断码，不允许断言直接终止进程。
- 工程容器只记录插件依赖元数据，不直接指定任意插件加载路径；插件仍由 `PluginHost` 统一扫描。
- 运行监视器、检查器和测试复用统一的错误码与部署诊断文本。

## 影响

- 运行时错误更稳定，可在 UI 中定位资源、输入格式和插件缺失问题。
- 部署要求从“隐式依赖本机 PATH”转为“显式诊断 Qt/OpenCV/插件 DLL 要求”。
- 新增的兼容性逻辑集中在迁移层和算法边界，不向 UI 扩散。
