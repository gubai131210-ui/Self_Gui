# ADR-0008：工程容器与执行快照边界

## 状态

已接受。

## 决策

工程容器使用目录格式：

- `manifest.json` 保存工程 ID、流程清单、资源清单和插件依赖。
- `flows/<flow-id>.json` 保存每个流程的图模型。
- `assets/` 保存随工程迁移的资源；资源引用使用相对路径。
- 保存先写随机临时目录，完成后再替换目标目录，并保留失败恢复用的备份。

执行开始时从当前 `Document` 创建 `FlowExecutionSnapshot`。运行时只读取快照，不读取随后被 UI 修改的工程对象。

`ExecutionScope` 支持完整流程、单节点、目标节点及上游依赖和显式子图。范围外的必需输入必须报告为“范围外缺少输入”，不能静默复用旧结果。

旧 `.selt` 继续由兼容层读取；容器迁移和资源缺失通过结构化报告提示，不阻塞查看工程图。

## 影响

容器格式暂不压缩，便于调试、版本迁移和资源诊断。插件依赖只由统一 `PluginHost` 解析，工程文件不能直接加载任意插件路径。
