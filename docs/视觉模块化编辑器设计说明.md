# 视觉模块化编辑器设计说明

## 定位

Selt Gui 当前定位为**机器视觉流程编辑器**：左侧视觉工具箱、中间流程画布、右侧模块参数、底部图像/结果视图。底层仍保留通用 Node / Canvas / Undo / 序列化能力，便于扩展，但主界面不再暴露思维导图类图形节点入口。

## 分层架构

| 层 | 目录 | 职责 |
|---|---|---|
| 算法层 | `vision/algorithms`, `vision/model` | 图像加载、灰度、二值、测量；ROI/叠加层/运行状态数据结构 |
| 集成层 | `vision/registry`, `vision/execution` | 模块元数据注册、参数校验、拓扑排序与单次执行 |
| UI 层 | `ui/*`, `graphics/*` | 工具箱、动态参数面板、画布节点状态、图像预览与 ROI 编辑 |
| 核心层 | `core/*` | Document、命令、序列化（与视觉解耦） |

设计模式：

- **Registry**：`VisionNodeRegistry` 统一注册模块描述（分类、固定尺寸、端口、参数定义）。
- **Builder**：`NodeBuilder` / `VisionNodeRegistry::create` 创建统一尺寸节点。
- **Command**：参数与节点变更走 `QUndoStack`。
- **Pipeline**：`VisionPipeline` 按依赖拓扑执行，缓存每模块输出与状态。

## 模块描述与参数

`ModuleDescriptor` 包含：

- `typeId` / `displayName` / `category` / `description`
- 固定节点尺寸（当前统一 `160×72`）
- 输入/输出端口
- `ModuleParamDef` 列表：`String` / `FilePath` / `Int` / `Double` / `Bool` / `Enum` / `Color` / `RoiRect`

右侧 `PropertyPanel` 按描述动态生成控件，不再按节点类型硬编码。参数写入 `NodeModel.parameters`，可撤销，并随 `.selt` 持久化。

后续可扩展参数绑定（上游输出/全局变量），当前仅本地参数。

## 执行模型

- `ExecutionMode::Once`：点击「单次执行」(F5) 后整图运行一轮并停止。
- 执行前做参数校验与拓扑排序；失败时记录 `failedNodeId`，画布高亮失败节点。
- `VisionContext.moduleResults` 缓存每模块输出图、叠加层、测量结果、耗时与状态。
- 预留后续：`单节点执行` / `从当前节点单步` / `断点`（通过扩展 `ExecutionMode` 与执行范围接口）。

模块状态：`Idle` / `Running` / `Success` / `Failed` / `Disabled`，同步到节点边框徽章与参数面板。

## 图像、ROI 与叠加层

- `ImagePreviewWidget`：原图 / 模块输出双视图能力，支持滚轮缩放、拖拽平移、适应窗口。
- ROI：`Ctrl + 拖拽` 绘制矩形 ROI，可清除；写回当前选中模块的 `roi` 参数。
- `OverlayItem`：矩形、轮廓、文字、线段；算法产出数据，UI 负责绘制，避免算法直接操作 Widget。

## 文件格式

- `format`: `selt-document`
- `version`: `3`（视觉流程参数仍存放在节点 `parameters` 中；v1/v2 可只读兼容加载）
- 旧思维导图节点仍可加载显示，但工具箱不再提供新建入口。

## 扩展新模块步骤

1. 在 `VisionNodeIds` 增加类型 ID。
2. 在 `VisionNodeRegistry` 注册 `ModuleDescriptor`（参数、端口、颜色）。
3. 在 `VisionPipeline::execute` 增加执行分支（后续可再抽象为 `VisionTool` 接口）。
4. 如有新算法，放入 `vision/algorithms` 并补充单元测试。
5. 需要时更新本说明与集成测试。

## 测试目标

- `Selt_Gui_tests`：文档/序列化核心
- `Selt_Gui_vision_tests`：算法层
- `Selt_Gui_vision_integration_tests`：注册表、参数校验、拓扑、单次执行、ROI/叠加层
